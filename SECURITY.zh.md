## 加密架构

Skerry 采用端到端加密，服务端零知识：

```
用户主密码 (Argon2id)
    ↓
主密钥 (masterKey)
    ↓
数据密钥 (dataKey, AES-256-GCM)
    ↓
加密每条记录: 主机配置、SSH 密钥、隧道设置等
```

### 密码学组件

| 组件 | 算法 | 用途 |
|------|------|------|
| 密钥派生 | Argon2id | 主密码 → masterKey |
| 数据加密 | AES-256-GCM | 每条记录独立加密 |
| 认证 | SRP-6a | 零知识登录，密码不上传 |
| JWT | HMAC-SHA256 | 会话令牌签名 |

### 零知识模型

- 服务器只存储加密后的数据块（ciphertext）和同步元数据（版本号、时间戳）
- 主密码仅在客户端内存中存活，永不上传
- SRP 认证仅验证"你知道密码"这一事实，不传输密码本身
- 配对码通过端到端加密传输加密的 dataKey，服务器无法解密
- 即使服务器完全被攻破，攻击者也只能获取加密后的字节流

### 本地数据存储

- 桌面端：加密保险库文件存储在 `%APPDATA%\Skerry` (Windows) / `~/.local/share/Skerry` (Linux) / `~/Library/Application Support/Skerry` (macOS)
- Android：应用私有存储，Android Keystore 保护
- 便携模式：检测到同级 `portable` 标记文件时，数据存储在程序目录

## 已知限制

- 内存中的密钥可能被具有 root 权限的进程读取（适用于所有密码管理器）
- 剪贴板操作后的密钥清除依赖系统剪贴板实现
- 桌面端无安全区域（Secure Enclave / TPM）集成 — 密钥仅在内存中

## 安全最佳实践

1. **主密码长度 ≥ 8 字符** — 越长越安全
2. **各设备使用不同主密码** — 降低单点泄露风险
3. **启用自动锁定** — 设置中配置空闲超时自动锁定
4. **定期审计已配对设备** — 管理控制台中检查并撤销不再使用的设备
5. **及时更新** — 确保使用最新版本获取安全修复
